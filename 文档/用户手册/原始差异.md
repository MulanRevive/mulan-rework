为了在重现过程中更易定位问题与更易开发维护，重现项目的部分功能与原始木兰可执行文件有区别。

- 部分报错信息改为中文包括表情符号等，但已知一些问题见 issue
- 支持中文命名标识符
- python 转换为木兰代码时，二元运算加括号以确保结果正确。如 2*(1+3)，原始木兰转换为 2*1+3
- python 转换为木兰代码时，部分语法节点输出不同。用原始木兰 `ulang-0.2.2.exe --python-to-ulang` 与重现项目 `木兰 --python变木兰` 对比：Python 源码 `x = {1, 2, 3}` 原始输出 `x = {1, 2, 3}`，重现项目输出 `x = 123`；`f = lambda x: x + 1` 原始输出 `f = (x) -> x + 1`，重现项目输出 `f = xx + 1`；`x = f'a{b}c'` 原始输出 `x = 'a' + str(b) + 'c'`，重现项目输出 `x = 'a'str(b)'c'`；含 `try/finally` 的示例原始输出包含 `finally { g() }`，重现项目只输出 `try { f() }`
- 引用（using）模块时，仅在木兰模块无法找到时退而寻找 python 模块；原始木兰中在引用木兰模块出错时不会报错而是直接寻找同名 python 模块
- 原始木兰使用系统默认编码读取源码，重现项目用UTF-8
- 命令行选项不同：原始木兰支持 `--help/-h`、`--dump-ast/-a`、`--exec-code/-e` 等英文选项；重现项目支持 `--帮助/-助`、`--语法树/-树`、`--执行代码/-执` 等中文选项。比如在重现项目中运行 `--help` 会提示“option --help not recognized”，需改用 `--帮助` 或 `-助`。另外，原始木兰支持 `--dump-blockly/-b` 生成 Blockly XML，重现项目目前没有对应命令行选项
- 生成木兰的 catch 时，重现项目成功；原始会报错 AttributeError: 'str' object has no attribute '_fields'
- 不同系统下测试发布版(v0.1.6)、开发版(v0.1.7)及原始木兰的最大递归深度不同：运行如下代码
  ```
  func test(a) {
      print(' '+str(a))
      test(a+1)
  }
  test(1)
  ```
  结果(最后一个输出)如下表
  | 系统                | Python版本 | 发布版结果 | 开发版结果 | 原始木兰结果 |
  |-------------------|----------|-------|-------|--------|
  | Win10             | 3.8.9    | 990   | 990   | 993    |
  | wsl(Ubuntu-22.04) | 3.10.6   | 993   | 990   | x      |
  | Macos             | 3.8.8    | 993   | 990   | x      |
