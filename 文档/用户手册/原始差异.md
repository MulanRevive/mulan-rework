为了在重现过程中更易定位问题与更易开发维护，重现项目的部分功能与原始木兰可执行文件有区别。

- 部分报错信息改为中文包括表情符号等，但已知一些问题见 issue
- 支持中文命名标识符
- python 转换为木兰代码时，二元运算加括号以确保结果正确。如 2*(1+3)，原始木兰转换为 2*1+3
- python 转换为木兰代码时，部分语法节点输出不同。用原始木兰 `ulang-0.2.2.exe --python-to-ulang` 与重现项目 `木兰 --python变木兰` 对比：Python 源码 `x = {1, 2, 3}` 原始输出 `x = {1, 2, 3}`，重现项目输出 `x = 123`；`f = lambda x: x + 1` 原始输出 `f = (x) -> x + 1`，重现项目输出 `f = xx + 1`；`x = f'a{b}c'` 原始输出 `x = 'a' + str(b) + 'c'`，重现项目输出 `x = 'a'str(b)'c'`；含 `try/finally` 的示例原始输出包含 `finally { g() }`，重现项目只输出 `try { f() }`
- python 转换为木兰代码时，`with`、`raise` 与按位取反的输出也不同。Python 源码 `with open("a") as f: x = f.read()` 中，原始木兰输出 `try f = open('a') { x = f.read() }`，重现项目输出 `open('a')f` 后跟 `x = f.read()`；源码 `raise RuntimeError("bad") from err` 中，原始木兰输出 `throw RuntimeError('bad')`，重现项目输出 `RuntimeError('bad')err`；源码 `x = ~a` 中，原始木兰输出 `x = (~a)`，重现项目会因 `ast.Invert` 抛出 `KeyError`
- python 转换为木兰代码时，`break` 和 `assert` 的输出也不同。Python 源码 `while a: break` 中，原始木兰输出 `while a { break }`，重现项目输出 `while a {}`；Python 源码 `assert x` 中，原始木兰输出 `assert (x`，重现项目只输出 `x`
- python 转换为木兰代码时，`continue`、`del` 与 `global` 的输出也不同。Python 源码 `while a: continue` 中，原始木兰输出 `while a { continue }`，重现项目输出 `while a {}`；Python 源码 `del x` 中，原始木兰输出 `unset x`，重现项目只输出 `x`；Python 源码 `def f(): global x; x = 1` 中，原始木兰输出的函数体包含 `extern x` 后跟 `x = 1`，重现项目只输出 `x = 1`
- python 转换为木兰代码时，推导式输出不同。Python 源码 `x = [i for i in a]` 中，原始木兰输出 `x = func() { __ = []; __.append(i)  for i in a; return __ }()`，重现项目输出 `x = iia`；源码 `x = {i: i for i in a}` 中，原始木兰输出通过临时函数填充字典，重现项目输出 `x = iiia`；源码 `x = {i for i in a}` 中，原始木兰输出通过临时函数填充 `set()`，重现项目输出 `x = iia`
- python 转换为木兰代码时，导入别名输出不同。Python 源码 `import os as operating_system` 中，原始木兰输出 `using os` 后跟 `operating_system = os`，重现项目只输出 `using os`；源码 `from os import path as p` 中，原始木兰输出 `using path in os` 后跟 `p = path`，重现项目只输出 `using path in os`
- python 转换为木兰代码时，`yield` 与注解赋值输出不同。Python 源码 `def f(): yield 1` 中，原始木兰输出的函数体包含 `yield 1`，重现项目只输出 `1`；源码 `x: int = 1` 中，原始木兰输出 `x : int = 1`，重现项目输出 `xint1`
- python 转换为木兰代码时，`nonlocal` 与函数可变参数输出不同。Python 源码 `def outer(): x = 1; def inner(): nonlocal x; x = 2` 中，原始木兰输出的内层函数体包含 `nonlocal x`，重现项目只保留 `x = 2`；源码 `def f(a, *args, **kwargs): return a` 中，原始木兰报错 `TypeError: can only concatenate str (not "arg") to str`，重现项目输出 `func f(a) { return a }`
- python 转换为木兰代码时，部分较新的 Python 语法处理不同。源码 `def f(a, /, b): return a` 在原始木兰中报 `SyntaxError: invalid syntax`，重现项目输出 `func f(b) { return a }`；源码 `if (x := f()): y = x` 在原始木兰中报 `SyntaxError: invalid syntax`，重现项目输出 `if xf() { y = x }`；含 `match x:`、`case 1:`、`y = 1` 的模式匹配源码在原始木兰中报 `SyntaxError: invalid syntax`，重现项目输出 `x1` 与 `y = 1`
- python 转换为木兰代码时，扩展切片输出不同。Python 源码 `x = a[1:3, 4]` 中，原始木兰报错 `TypeError: cannot unpack non-iterable Slice object`，重现项目输出 `x = a[tuple(1:3, 4)]`
- python 转换为木兰代码时，带步长的切片输出不同。Python 源码 `x = a[1:10:2]` 中，原始木兰输出 `x = a[1:10:2]`，重现项目输出 `x = a[1:10]`
- 引用（using）模块时，仅在木兰模块无法找到时退而寻找 python 模块；原始木兰中在引用木兰模块出错时不会报错而是直接寻找同名 python 模块
- 原始木兰使用系统默认编码读取源码，重现项目用UTF-8
- 源码文件第一行以 shebang（`#!...`）开头时，原始木兰会忽略这一行继续执行；重现项目会报语法错误。例如如下源码：
  ```
  #!/usr/bin/env ulang
  println(1)
  ```
  原始木兰 `ulang-0.2.2.exe 文件.ul` 输出 `1`，重现项目 `木兰 文件.ul` 报错 `没认出这个词 "#"`。
- 命令行直接执行代码时，`__file__` 的值不同：原始木兰中 `ulang --exec-code="println(__file__)"` 输出 `<CLI>`；重现项目中 `木兰 --执行代码="println(__file__)"` 输出 `<命令行>`
- 命令行选项不同：原始木兰支持 `--help/-h`、`--dump-ast/-a`、`--exec-code/-e` 等英文选项；重现项目支持 `--帮助/-助`、`--语法树/-树`、`--执行代码/-执` 等中文选项。比如在重现项目中运行 `--help` 会提示“option --help not recognized”，需改用 `--帮助` 或 `-助`。另外，原始木兰支持 `--dump-blockly/-b` 生成 Blockly XML，重现项目目前没有对应命令行选项
- 命令行运行环境中的内置值不同：执行命令行代码时，`println(__file__)` 在原始木兰中输出 `<CLI>`，在重现项目中输出 `<命令行>`；从标准输入读取源码时，原始木兰输出 `<STDIN>`，重现项目输出 `<标准输入流>`。另外，执行命令行代码并继续传入参数时，`println(ARGV)` 的范围也不同：原始木兰运行 `--exec-code=println(ARGV) a b` 输出 `[b]`，重现项目运行 `--执行代码=println(ARGV) a b` 输出 `[a, b]`；直接执行源码文件时未见这个 `ARGV` 差异。
- 生成木兰的 catch 时，重现项目成功；原始会报错 AttributeError: 'str' object has no attribute '_fields'
- 无参匿名函数 `func { ... }` 在重现项目中可作为无参函数调用；原始木兰会报错 `TypeError: required field "args" missing from arguments`。例如：
  ```
  f = func { println(42) }
  f()
  ```
  重现项目输出 `42`，原始木兰无输出并报上述错误。
- 对字面量使用严格相等 `===` 或严格不等 `!==` 时，原始木兰只输出比较结果；重现项目在 Python 3.13.0 下会额外产生 `SyntaxWarning`。例如 `println(1 === 1); println(1 !== 2)` 两边都输出 `true`、`true`，但重现项目还会提示 `"is" with 'int' literal` 与 `"is not" with 'int' literal`
- 不同系统下测试发布版(v0.1.6)、开发版(v0.1.7)及原始木兰的最大递归深度不同：运行如下代码
  ```
  func test(a) {
      print(' '+str(a))
      test(a+1)
  }
  test(1)
  ```
  结果(最后一个输出)如下表
  | 系统                | Python版本 | 发布版结果 | 开发版结果 | 原始木兰结果 |
  |-------------------|----------|-------|-------|--------|
  | Win10             | 3.8.9    | 990   | 990   | 993    |
  | wsl(Ubuntu-22.04) | 3.10.6   | 993   | 990   | x      |
  | Macos             | 3.8.8    | 993   | 990   | x      |
